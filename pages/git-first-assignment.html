Using Git for the first assignment

The first assignment requires minimal knowledge of Python and some knowledge of Linux/Unix. (Knowledge of Unix is an official pre-requisite for this course.) Those wishing to learn how to work in the Unix environment should check out the following website, The Unix Shell. The IT department at the university also has a lengthy pdf tutorial, Unix: The Basics.

These instructions assume you have created a GitHub account.

Follow the GitHub Classroom Link

Following the GitHub Classroom link posted on Canvas should leave you with a private repository with a name like "z639-2021/assignment1-{username}", where {username} is your GitHub username. So an example of a repository would be z639-2021/assignment1-myusername, accessible to its owner at https://github.com/z639-2021/assignment1-myusername.

Configure SSH authentication

The following steps assume you have connected to one of the School Linux servers (e.g., gh.luddy.indiana.edu) and that you have not already generated an SSH key. If anything is confusing, consider reading GitHub's own version of these instructions, Connecting to GitHub with SSH.

  1. Generate a new SSH key: ssh-keygen -t ed25519. This will create two files in the ~/.ssh directory: ~/.ssh/id_ed25519 and ~/.ssh/id_ed25519.pub.
  2. Add the SSH public key to your GitHub account: Add the contents of ~/.ssh/id_ed25519.pub to your GitHub account (look under Settings, under "SSH and GPG keys").

That's it. You can test that things have been configured with ssh -T git@github.com. If you configured things correctly, you'll see a message with your username.

Clone the repository

Clicking on the Code button will let you create a local copy of the repository. Use the Clone with SSH option for now. You should be given instructions about what URL to use with git. For example, it might tell you to clone the following address, git@github.com:z639-2021/assignment1-myusername.git. Cloning this repository means running the following command in the Unix environment:

git clone git@github.com:z639-2021/assignment1-myusername.git

You should have a directory in your current directory called assignment1-{username} (e.g., assignment1-myusername). You can cd to this directory and look around inside using ls.

Read the instructions in the README.md file. Some assignments have special instructions

Edit the assignment file directly using an editor

Edit assignment1.py in a text editor such as nano, vi, or emacs. Replace the sections marked # YOUR CODE HERE with Python code. A description of what the function should do is contained in the function's docstring.

Check that your assignment is free from syntax errors and indentation errors

Before submitting your code, check that your assignment is free of trivial Python errors. You can do this by executing the file: python3 assignment1.py. If you've made a syntax error Python will report this by printing SyntaxError: invalid syntax. If there are indentation errors, Python will tell you. Python should also tell you on what line an error occurs. Fix these errors first.

If you don't have any syntax errors, a couple of superficial tests will be run on your code to see if your functions are working. These tests check to see if you're on the right track. They are not exhaustive tests. On your very first assignment you may wish to ignore these entirely. Just make sure that you don't have any syntax or indentation errors.

Submit your assignment

When you are ready to submit your assignment you need to push it to the GitHub server so the autograder can grade it. First add your modified file with

 git add assignment1.py

Now you need to commit your changes. You do need to provide a "commit message" in order to do this. The commit message does not matter. (When collaborating on code with other people, informative commit messages are essential.)

git commit

Now push your changes (i.e., upload them):

git push

You're done.
