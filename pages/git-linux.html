Git at IU

Note: the following assumes you have read Using Python on IU Linux Computers.

This document describes how to commit changes to a file and how to push committed changes to a remote git server (e.g., Gitlab or Github). The following assumes you have read Using Python on IU Linux Computers and have superficial knowledge of how to use git.

Cloning a remote repository

You can create a copy of a remote repository locally using git clone. To clone assignment 2 from Github you would type something resembling:

git clone https://github.com/z639-2021/assignment2-myusername.git

You will be prompted for your username and password. If entering your username and password gets tiring, you can run this command once: git config --global credential.helper store. (Experienced users may wish to setup SSH authentication).

Making changes to a file

Note: you can always type git status to see what changes, if any, are pending.

Text files may be edited using any one of the standard text editors such as nano, vi, or emacs. (You can also edit files offline and upload them using sftp.)

Once you have made changes to a file, add and commit the changes with git add and git commit. For example, you can add and commit changes to the file assignment2.py with the following commands:

git add assignment2.py
git commit

After running git commit you will be prompted to enter a short description of your changes.

Pushing your changes to the remote repository

Tell the remote repository about your committed changes by using git push.
